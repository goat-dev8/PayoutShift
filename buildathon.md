Enhanced UX & Reliability

This wave introduced four major features that dramatically improve the bot's usability, intelligence, and safety.

1. Voice Interface Implementation
Users can now send a voice message instead of typing. The bot transcribes the audio using Groq's Whisper API and processes the resulting text command.

- bot.ts
  - Dependencies: Added axios (for downloading the file), fs (for saving temp files), path (for file paths), and execSync (for running ffmpeg).
  - FFmpeg Check: On startup, the bot now checks if ffmpeg is installed on the server. If not, it logs a warning, as voice messages will fail.
  - New Handler: Added bot.on(message('voice'), ...) to catch all voice messages.
  - Logic:
    1. The handler gets the file_id from ctx.message.voice.file_id.
    2. It calls ctx.telegram.getFileLink(file_id) to get a private download URL for the .oga file.
    3. axios downloads the audio file as a buffer.
    4. fs.writeFileSync saves the buffer to a temporary file (e.g., temp_12345.oga).
    5. execSync calls ffmpeg -i temp_12345.oga temp_12345.mp3 -y to convert the file to MP3, which the transcription API requires.
    6. The new transcribeAudio function is called with the MP3 file path.
    7. The bot replies with I heard: "..." and passes the transcribed text to the handleTextMessage function, unifying the command logic for both text and voice.
    8. Finally, fs.unlinkSync cleans up both the .oga and .mp3 temp files.

- groq-client.ts
  - New Function: Added an async exported function transcribeAudio(mp3FilePath).
  - Logic: This function uses groq.audio.transcriptions.create() with the "whisper-large-v3" model to get the transcription from the provided MP3 file.

2. Transaction Tracking & Status Updates
Users can now check the status of their swaps at any time using the /status command.

- sideshift-client.ts
  - New Function: Added getOrderStatus(orderId).
  - Logic: This performs an authenticated GET request to the https://sideshift.ai/api/v2/shifts/:orderId endpoint.
  - New Type: Added the SideShiftOrderStatus interface to strongly type the JSON response from the API.

- database.ts
  - Schema Change: Modified the CREATE TABLE IF NOT EXISTS orders statement.
    - Added a tx_hash TEXT column to store the user's outbound transaction hash.
    - The status column was changed to a generic TEXT to accommodate various statuses from the API (e.g., waiting, confirming, settled).
  - New Functions:
    - getLatestUserOrder(telegramId): Fetches the most recent order for a user, used by /status if no ID is provided.
    - updateOrderStatus(sideshiftOrderId, newStatus): Updates the status field for an order in our local DB.
    - setOrderTxHash(sideshiftOrderId, txHash): Populates the new tx_hash column after a user sends their transaction.
